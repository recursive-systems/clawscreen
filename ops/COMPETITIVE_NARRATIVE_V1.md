# Competitive Narrative v1 (vs Chat-only Assistants)

## Core framing
Chat-only assistants are great for answers.
ClawScreen is for outcomes.

## Where chat-only stalls
- Context is shallow and session-fragile
- Actions are implicit ("I would do X") instead of executed
- No durable workflow state for teams
- Hard to observe, debug, and iterate in production

## What ClawScreen adds
- **Actionable UI primitives**: visible steps, state, and execution paths
- **Human-in-the-loop controls**: approval points, edits, overrides
- **Operational feedback loop**: logs, traces, retries, guardrails
- **Product-native integration**: ships as part of your app, not beside it

## Competitive posture
- Not anti-chat: chat is an entry point
- Post-chat architecture: conversation -> controlled execution
- Moat direction: reliability + observability + embedded workflow UX
